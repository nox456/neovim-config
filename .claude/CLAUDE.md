# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A personal Neovim configuration built on **NvChad v2.5**, tuned for web dev (TypeScript/React), Go and a Git-heavy workflow. It is pure Lua config — there is no build step and no test suite. "Running" it means launching `nvim`; changes are verified by reloading Neovim, not by a CLI.

## Commands

- **Format Lua** (the only linter/formatter that applies to this repo's own code): `stylua .` — settings in `.stylua.toml` (2-space indent, 120 col, double quotes, no call parentheses). Match this style in any `.lua` you edit.
- **Apply config changes:** restart Neovim, or `:Lazy sync` after editing plugin specs in `lua/plugins/init.lua`.
- **Diagnose a broken setup:** `:checkhealth`. Plugin issues: `:Lazy`. LSP/formatter/tool installs: `:Mason`.
- Note: the formatters/linters configured under `lua/configs/` (`prettierd`, `biome`, `eslint`, `ts_ls`, `gopls`, …) act on **files the user edits**, not on this config. They are only relevant when reasoning about the user's editing experience.

## Load order (init.lua)

`init.lua` is the single entry point and the order is significant:

1. Sets `vim.g.mapleader = ","` **before** anything else (leader is comma, not space).
2. Bootstraps `lazy.nvim` (clones it on first run), then `lazy.setup` loads `NvChad/NvChad` (branch `v2.5`, which imports `nvchad.plugins`) followed by everything under `lua/plugins/`.
3. Sources the precompiled **base46** theme cache from `stdpath("data")/base46/` (`defaults` + `statusline`). This cache is generated by NvChad — theme/highlight changes only take effect after the cache is rebuilt (re-run via NvChad, e.g. switching themes with `:lua require('nvchad.themes').open()`).
4. `require "options"` → `require "autocmds"` → (scheduled) `require "mappings"`.

## Architecture / where things live

This config **layers on top of NvChad** rather than replacing it. Each entry file first calls into the NvChad equivalent, then adds overrides:

- `lua/options.lua` → `require "nvchad.options"` then custom vim options.
- `lua/autocmds.lua` → `require "nvchad.autocmds"` then custom autocmds.
- `lua/mappings.lua` → `require "nvchad.mappings"` then custom keymaps.
- `lua/chadrc.lua` → must keep the shape of NvChad's `nvconfig.lua`; this is where the theme, the `nvdash` dashboard buttons, and the **custom arrow-style statusline** (`M.ui.statusline`, with hand-written `filename`/`filepath`/`file_position` modules) live.

Plugin specs added on top of NvChad go in `lua/plugins/init.lua`. Plugin-specific config that's long enough to extract lives in `lua/configs/` (`lazy.lua`, `lspconfig.lua`, `conform.lua`) and `lua/nvchad/configs/nvimtree.lua`, referenced from the spec via `require`/`opts`.

### Adding to the config

- **A plugin:** add a spec to `lua/plugins/init.lua`, then `:Lazy sync`.
- **An LSP server:** append its name to the `servers` list in `lua/configs/lspconfig.lua` and install it with `:Mason`.
- **A formatter:** add it under `formatters_by_ft` in `lua/configs/conform.lua`.
- **A keymap:** add it to `lua/mappings.lua`. The full custom-keymap reference is `lua/KEYBINDINGS.md` — keep it in sync when you change mappings.

## Things that will bite you

- **Leader is `,`** (comma), set at the top of `init.lua`. Don't assume space.
- **Format-on-save is intentionally off** — the `format_on_save` block in `lua/configs/conform.lua` is commented out; formatting is manual via `<leader>i`. Don't uncomment it without being asked.
- **`nvim-lint` is intentionally disabled** (`enabled = false` in its spec) even though an eslint-for-TS config is staged there.
- **snacks main-window fix:** an autocmd in `lua/autocmds.lua` marks the `nvdash` dashboard buffer with `snacks_main = true`. Without it, snacks' file pickers fall back to opening files in special windows (the dashboard or the Claude Code terminal). Be careful editing the dashboard/terminal-related autocmds and mappings.
- **Claude Code keymap (`<leader>cc`)** opens a vertical terminal and `chansend`s `clear && claude && exit` into it.
- **`undofile` is off** and folds default to fully open (`foldlevel`/`foldlevelstart = 99`, folding via nvim-ufo).
- `lazy-lock.json` is the committed version lockfile — treat changes to it as deliberate dependency bumps.
